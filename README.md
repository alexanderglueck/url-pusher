# URL-Pusher

> Instantly push URLs to your devices

## About URL-Pusher

URL-Pusher allows you to push URLs from your browser to any of your Android
phones instantaneously. URLs are delivered to registered devices through
Firebase Cloud Messaging.

## Tech stack

- **Laravel 13** (PHP 8.4+) on the slim application skeleton
- **Inertia.js + Vue 3** for the front-end
- **Tailwind CSS v4** (CSS-first configuration, no `tailwind.config.js`)
- **Vite** for asset bundling
- **Laravel Jetstream** (Inertia stack) + **Fortify** for authentication
- **Laravel Sanctum** for the token-based mobile API
- **Scramble** for the auto-generated OpenAPI specification
- **kreait/laravel-firebase** for Firebase Cloud Messaging

## Local development

The project ships with a Docker setup. The `app` image is a multi-stage build
(`production` / `dev` / `testing` targets); `docker-compose.yml` uses the `dev`
target with the source bind-mounted.

```bash
# Build and start the containers (nginx + php-fpm + mysql)
docker compose up -d --build

# Install dependencies (run inside the app container)
docker compose exec app composer install
npm install

# Application key, migrations
docker compose exec app php artisan key:generate
docker compose exec app php artisan migrate

# Front-end dev server (Vite)
npm run dev
```

The app is served at <http://localhost:8088>.

### Useful commands

```bash
docker compose exec app php artisan test   # run the test suite
docker compose exec app composer pint      # format PHP code
npm run build                              # build production assets
```

## API

The mobile API lives under `/api/v1`, is authenticated with Laravel Sanctum
tokens, and reuses the same domain logic as the web application (shared action
classes). The OpenAPI specification is generated by Scramble and served at
`/docs/api`.
